BFLFP Flow CORS Test

This page verifies whether the Power Automate flow can be called from this origin (). Run it from the deployed GitHub Pages URL before assuming the dashboards will work.

What to look for

  • ✅ Success: Status 200 and a JSON response body. CORS is fine.
  • ❌ CORS error: "blocked by CORS policy" in console. The flow's Response action needs Access-Control-Allow-Origin: * header. Edit the flow → Response action → Add header.
  • ❌ 401/403: Signature is wrong or expired. Regenerate the flow URL.
  • ❌ 400 on POST + 405 on GET: The trigger method is restricted. Allow both POST and GET in the flow trigger.

How to fix CORS in Power Automate

  1. Open the flow in Power Automate
  2. Click the Response action (last step)
  3. Click Show advanced options
  4. In the Headers field, add: 
    {
  "Content-Type": "application/json",
  "Access-Control-Allow-Origin": "*",
  "Access-Control-Allow-Methods": "GET, POST, OPTIONS",
  "Access-Control-Allow-Headers": "Content-Type"
}
  5. Save the flow and retest here
